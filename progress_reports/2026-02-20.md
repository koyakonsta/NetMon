We decided to scan the network activity using `tcpdump`.
We were originally planning to do this using a VPNService, but after research decided against it, as using `tcpdump` happened to be a cleaner solution.
It captures all network packets that the device receives, and the app analyses the headers to detect any abnormal behaviour.

The `tcpdump` process runs in background. The app then analyses the receives packets to detect network and port scans as well as ARP poisoning attacks.

We have chosen to have the device list show up on the main page, with all information about the device being displayed there.
For any device where unusual activity was detected, there is a list of detected attacks below it.

There is a log file that contains the list of all threats that have been detected. It is accessible via the "Logs" page.
The file contains some further detail about each attack, such as the devices that were connected to (for network scans) or address information (for ARP spoofing attacks).
