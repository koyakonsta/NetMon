We have found a handful of apps that achieve similar goals as our project, such as Fing, PingTools, and ARP Spoof Detect : Wifi Guard.
- Fing covers a fairly broad set of features, some of which are similar to Network Monitor's:
   - Lets the user see the devices connected to the network and allows the user verify the security of their local network
   - Alerts users whenever a new device connects to the network
   - Helps users detect and troubleshoot network issues
   - Lets users limit devices' Internet access to specified schedules
   - However, it doesn't appear to cover ARP spoofing
- PingTools also provides some network utilities:
   - Lists devices connected to the network
   - Shows information about the network
   - Monitors resource connectivity
   - Pings resources, using ICMP, TCP, and HTTP pings
   - Doesn't seem to have any automated scans
- ARP Spoof Detect : Wifi Guard detects ARP spoofing and poisoning attacks:
   - Regularly scans the network to detect ARP attacks
   - Shows details about the last attack, and shows a log of previous attacks
   - Doesn't include any other features

Our Network Monitor project will do the following:
- Regularly scan the network for all connected devices
- Store information (user-provided device name, device type, IP, MAC) about trusted devices
- Alert user about unrecognised devices or changes to a trusted device
- Detect ARP spoofing and ARP cache poisoning attacks

For the project work, we have found Chaquopy, which is a Python SDK for Android. This will allow us to write most of the application code in Python, but I included Java in the list of programming languages in the proposal in case there's anything that might require being implemented in Java.

We have not yet identified use cases for the functional spec, but we will work on that next.

For the demo, we will need to simulate a LAN with devices connected to it. Not sure how we will do that at this stage yet.
